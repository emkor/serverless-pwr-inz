Technical notes

Domain and purpose

The app retrieves information helpful for potential travellers from public APIs of 3rd parties like Weather API, Uber API, Instagram API and SkyScanner API.

App is focused on western-European countries, but adding more cities would not be a problem.

Application was built to research possibilities of Serverless architecture approach and confront it with classic distributed approach.

Frameworks and technologies used

Front-end application is written using angularjs framework with bootstrap applied and based on node.js platform while running on npm's built-in http-server.

Backend API is written in Python 2.7 with usage of Celery framework (with RabbitMQ as a message broker and MySQL as result storage) for local executor and is exposed by CherryPy web server. AWS Lambda executor uses Serverless Framework for deploying remote functions.

For deployment of each of the component (excluding AWS Lambda executor), Docker container is used. Managing deployment is possible thanks to Docker Compose technology.

Background

Application was created as a part of Engineering Thesis at Wroclaw University of Technology (Faculty of Management and Computer Science) by Mateusz Korzeniowski.

Thesis subject is: Web based application supporting planning of tourist travel, built using Serverless architectural pattern
